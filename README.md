# To_Do_List
Projeto que visa auxiliar o usuário a organizar suas tarefas do dia a dia, permitindo o gerenciamento completo de atividades de forma simples e segura.
## Visão Geral
O To-Do List é um projeto desenvolvido com o objetivo de praticar e consolidar conceitos fundamentais do desenvolvimento fullstack.

O foco principal foi a aplicação de:
 - Arquitetura em camadas
 - Injeção de dependência
 - Boas práticas de versionamento (Git e GitHub)
 - Autenticação e autorização utilizando JWT
 - Validação de requisições no middleware
 - Documentação de projeto
 - Separação clara de responsabilidades entre frontend e backend
## Tecnologias Utilizadas 
### BackEnd
 - C#
 - ASP.NET Core Web API
 - Entity Framework Core
 - MySql
### FrontEnd
 - Angular 
 - Typescript
 - Bootstrap
### Outros
 - Git 
 - GitHub
## Funcionalidade Principais 
 - Cadastro de usuário
 - Login com autenticação via JWT
 - Armazenamento do token no frontend
 - Cadastro de tarefas
 - Edição de tarefas
 - Conclusão de tarefas
 - Exclusão de tarefas
 - Visualização apenas das tarefas do usuário autenticado
## Como execultar o projeto localmente
### Pré requisitos
 - .NET SDK 6+
 - Angular CLI 15
 - MySQL 
### BackEnd
 - Clone o repositorio 
 - Instalar as dependencias do projeto
 - Configure a string de conexão com o Banco de Dados
 - Execute as migrations 
 - Inicie a API
### FrontEnd
 - Acesse a pasta do frontend
 - Instale as dependencias 
 - Inicie a aplicação Angular
## Considerações Finais
Este projeto foi desenvolvido com foco em boas práticas, organização de código e segurança, simulando um cenário próximo ao de aplicações reais utilizadas em produção.
